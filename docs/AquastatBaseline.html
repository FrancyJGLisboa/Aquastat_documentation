Migration of AQUASTAT Statistical Processes into the SWS

Chapter 1 The faoswsAquastatBaseline module

1.1 Aim

According to the technical unit, there is a set of AQUASTAT variables that should have the same value over time in the aquastat_legacy dataset. These variables are collectivelly known as Long-term average variables. However, it was observed that, for an unidentified reason, some of these variables have wrong values in the old AQUATSAT Working System and these errors persisted after the migration from the old system into SWS aquastat_legacy dataset. Therefore, the faoswsAquastatBaseline module has the task of correcting these wrong values.

1.1.1 Identify LTAs

The very first thing the module needs to do is to identify the LTAs in the aquastat_legacy dataset. To identify the LTAs, the faoswsAquastatModule uses the SWS aquastat_reference data table. This data table has as variable called ‘lta’ which has value 1 for the LTA variables/indicators and 0 for non-LTAs. The table below describes the LTAs in the SWS aquastat_reference data table.

1.1.3 Corrections

With the emergence of right LTA values, the module corrects the LTAs by equating their observed values in each geographicAreaM49 - aquastatElement combination.

1.3 Running the module

  1. Log in the SWS;

  2. Click on New Query;

  1. Select AQUASTAT domain and aquastat_baseline dataset;

  2. Select whatever country, measured element, and timePointYears;

The faoswsAquastatAquastatBaseline corrects the WHOLE aquastat_legacy dataset which means that the user’s selections during this step are irrelevant for the module’s output.

  1. Run the query;

  2. Click on Run plugin on the top-right.

  3. Select the faoswsAquastatExternal module and click on Run plugin;

Figure 1.2: Select Aquastat baseline module

  1. Wait for the results to appear in the session;

  2. Click on Save to dataset after validation.

1.5 Output feeds…

The corrected version of the aquasat_legacy dataset output by the faoswsAquastatBaseline module is the aquastat_baseline dataset. This dataset serves as an input of the faoswsAquastatUpdate.



The aquastat_baseline keeps the same dimension names as the legacy dataset: geographicAreaM49, aquastatElement, timePointYears, and flagAquastat.

The change of aquastatElement to measuredElement (SWS compliant) as well as the flag conversion from flagAquast to flagObaservationStatus are tasked to the faoswsAquastatUpdate module.
